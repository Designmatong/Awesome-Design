##  Why Awesome Design?

The way we build interfaces is changing. AI coding agents can now generate production-quality UI — but only when they understand **how it should look**.

**Awesome Design** is a curated ecosystem of design resources purpose-built for the AI era. Every file here is crafted to be read by both humans and machines — plain-text design systems that turn AI agents into design-aware builders.

> **Copy a file. Tell your agent. Ship consistent UI.**
>
> No Figma exports. No JSON schemas. No special tooling.
> Just markdown — the format LLMs read best.

### Quick Start

```bash
# 1. Clone this repo
git clone https://github.com/Designmatong/Awesome-Design.git

# 2. Pick a design language
ls awesome-design-md/design-md/

# 3. Copy it to your project
cp Awesome-Design/awesome-design-md/design-md/stripe/DESIGN.md ./my-app/DESIGN.md

# 4. Let your AI agent work
# "Build a landing page using the design system in DESIGN.md"
```

##  Design Philosophy

<div align="center">
  <br/>

  | Principle | Meaning |
  |:---------:|---------|
  | **Plain Text First** | Markdown is the universal language of LLMs. No proprietary formats. |
  | **Tokens, Not Screenshots** | Semantic design tokens that code can reference — not static mockups. |
  | **Extracted, Not Invented** | Every DESIGN.md comes from a real, live website. Authentic systems. |
  | **Agent-Native** | Designed from the ground up to be read, understood, and applied by AI. |

##  Contributing

We welcome contributions that make design systems more accessible to AI agents.

- **Improve a DESIGN.md** — Spot a wrong hex value or missing token? [Open an issue](https://github.com/Designmatong/Awesome-Design/issues) first, then submit a PR.
- **Suggest a site** — Know a product with a distinctive design language? Let us know.
- **Spread the word** — Star the repo, share with your team, help grow the collection.

See [CONTRIBUTING.md](awesome-design-md/CONTRIBUTING.md) for detailed guidelines.
