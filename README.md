##  为什么选择 Awesome Design？ · Why Awesome Design?

我们构建界面的方式正在发生根本性变化。AI 编程智能体已经能够生成生产级 UI——但前提是它们必须理解**界面应该长什么样**。

**Awesome Design** 是一个为 AI 时代精心打造的设计资源生态系统。这里的每个文件都经过精心设计，同时面向人类和机器——纯文本设计系统，将 AI 智能体转变为具备设计感知力的构建者。

> **复制一个文件。告诉你的 AI 智能体。交付一致的 UI。**
>
> 不需要 Figma 导出。不需要 JSON schema。不需要特殊工具。
> 只有 Markdown——LLM 阅读效果最好的格式。

---

The way we build interfaces is changing. AI coding agents can now generate production-quality UI — but only when they understand **how it should look**.

**Awesome Design** is a curated ecosystem of design resources purpose-built for the AI era. Every file here is crafted to be read by both humans and machines — plain-text design systems that turn AI agents into design-aware builders.

> **Copy a file. Tell your agent. Ship consistent UI.**
>
> No Figma exports. No JSON schemas. No special tooling.
> Just markdown — the format LLMs read best.

##  设计哲学 · Design Philosophy

<div align="center">
  <br/>

  | 原则 Principle | 含义 Meaning |
  |:-------------:|-------------|
  | **纯文本优先**<br/>**Plain Text First** | Markdown 是 LLM 的通用语言。无私有格式。<br/>Markdown is the universal language of LLMs. No proprietary formats. |
  | **Token 代替截图**<br/>**Tokens, Not Screenshots** | 代码可直接引用的语义化设计 token——而非静态效果图。<br/>Semantic design tokens that code can reference — not static mockups. |
  | **提取而非臆造**<br/>**Extracted, Not Invented** | 每个 DESIGN.md 源自真实的线上网站。真实的设计系统。<br/>Every DESIGN.md comes from a real, live website. Authentic systems. |
  | **AI 原生**<br/>**Agent-Native** | 从底层设计即面向 AI 的阅读、理解与应用。<br/>Designed from the ground up to be read, understood, and applied by AI. |
